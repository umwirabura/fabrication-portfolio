## ğŸ“ Documentation Guidelines

### Adding Content

1. Navigate to the appropriate section in `docs/`
2. Edit or create markdown files
3. Add images to `docs/images/[section]/`
4. Add downloadable files to `docs/files/[section]/`
5. Update navigation in `mkdocs.yml` if adding new pages

### Markdown Tips

- Use `#` for headings (# H1, ## H2, ### H3)
- Add images: `![Alt text](images/section/filename.png)`
- Create links: `[Link text](page.md)`
- Use admonitions for callouts:
  ```markdown
  !!! note "Title"
      Content here
  ```

### Commit Conventions

- Use clear, descriptive commit messages
- Present tense: "Add PCB design" not "Added PCB design"
- Keep commits focused on single topics
